The only dynamic portion of the project is that you can include links to your archives and other pages using JSON file includes. ### Writing a post This is a really simple setup. You also have options. The basic way to create a new post, however, is to create a new file by copying & pasting the template file. You can add new content in HTML or use markdown by writing within a div (or similar element) with the class `marked`. ### Structure There are options. The default procedure is to simply save each new post as `index.html`. This means that the main page shows the most recent article. When it is time to write a new article, rename the file for archiving and create a new post with the name `index.html`. The code snippet for adding a list of recent articles to the bottom of the page will autopopulate with the most recent articles. You could write a new script to pull only those articles that are relevant to the current topic. For the hackers, the options are almost limitless. If you create a new structure, create a pull request or file an issue so that I can add more options for users. I will be working on a generic blog setup where the most current post shows up at the top of the main page, but you'll have to wait for that functionality to be added (or create it yourself!). ### Markdown Thanks to [marked.js](https://github.com/chjj/marked) (v 0.3.7) you can use markdown instead of writing HTML. This helps you to focus on your content. However, if you don't like markdown or don't know it yet, you can use HTML without any restrictions. To use the markdown feature, it is very simple: * Wrap your markdown in a `<div>` (or similar element) and give it the class "marked". * Be sure not to indent your markdown too far- it will default to `<pre><code></code></pre>` if you do and won't render. * Optional: if you want to use markdown within an HTML element, wrap it in a `<span>` (I have included a script that strips the `<p>` tag when you use a `<span>`) To learn more about writing in markdown, view [Jon Schlinkert's](https://gist.github.com/jonschlinkert) [cheatsheet](https://gist.github.com/jonschlinkert/5854601) ### JS Includes To use the includes, you may want to learn a little about JSON (if you don't already). You don't need to be an expert and can very likely get away without learning it all, but why not learn something new? Check out these [JSON](https://www.json.com/) and [JSON Array](https://www.json.com/json-array) pages. Data for your links is stored in JSON arrays and then linked to the page template, so that with each new page you create, the links will automatically be included. However, this is an extra step- you'll need to update the JSON files when you write a new post. It's pretty simple: * Open the posts.js file for editing * Copy the last entry- it should look something like this: ``` { "title": "JSON Overview", "url": "https://www.json.com", "date":"5/5/15" } ``` * Add a comma after this last curly brace * Paste the copy on a new line after the last curly brace and comma * Change the info to reflect your new post * Commit (save) the changes to the posts.js file The process for creating and updating the external links is the same except that you'll be dealing with the links.js file instead. If you plan to hack this project, you can certainly include more metadata- just be sure to use proper JSON syntax. ### Use & License This work is free to use with an MIT license.
